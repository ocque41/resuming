## 7. AWS Production Configuration

### 7.1. AWS Console Access and Navigation
- [ ] AWS Management Console Access:
  - [ ] Open web browser and navigate to https://aws.amazon.com/
  - [ ] Click "Sign In to the Console" in the top-right corner
  - [ ] Enter AWS account credentials
  - [ ] Use multi-factor authentication if configured
  - [ ] Select appropriate AWS region from the dropdown in the top-right corner (e.g., us-east-1)
- [ ] AWS Console Navigation:
  - [ ] Use the search bar at the top to find services quickly
  - [ ] Use the "Recently visited" section for frequently used services
  - [ ] Add important services to favorites by clicking the star icon
  - [ ] Learn keyboard shortcuts (press '?' in the console to view)

### 7.2. Lambda Service Setup
- [ ] Access Lambda Console:
  - [ ] From AWS Management Console, search for "Lambda" and select it
  - [ ] Click "Create function" button to start the function creation process
- [ ] Create Lambda Function:
  - [ ] Select "Author from scratch" option
  - [ ] Enter function name following naming convention "ai-document-agent-[function-purpose]"
  - [ ] Select Python 3.9 or 3.10 as runtime
  - [ ] Under "Permissions", select "Create a new role with basic Lambda permissions"
  - [ ] Click "Create function" button
- [ ] Configure Lambda Function Settings:
  - [ ] In the "Configuration" tab, click "General configuration" and edit
  - [ ] Set Memory to at least 512 MB (or higher based on requirements)
  - [ ] Set Timeout to 30 seconds (adjust based on function needs)
  - [ ] Enable "Configure function URL" if direct HTTP access is needed
  - [ ] Click "Save" to apply changes
- [ ] Create Lambda Service Role:
  - [ ] Navigate to IAM console (search for "IAM" in the AWS console)
  - [ ] Click "Roles" in the left sidebar, then "Create role"
  - [ ] Select "AWS service" as trusted entity type, then "Lambda" as use case
  - [ ] On the "Add permissions" page, search for and select the following policies:
    * AmazonS3ReadOnlyAccess (or create custom S3 policy)
    * AmazonDynamoDBFullAccess (or create custom DynamoDB policy)
    * AWSLambdaBasicExecutionRole (for CloudWatch Logs)
    * CloudWatchLogsFullAccess (for more detailed logging)
  - [ ] Name the role "ai-document-agent-lambda-role" and add description
  - [ ] Click "Create role" button
- [ ] Assign Role to Lambda Function:
  - [ ] Return to Lambda function in the console
  - [ ] Select "Configuration" tab, then "Permissions"
  - [ ] Click "Edit" next to "Execution role"
  - [ ] Select "Use an existing role" and choose "ai-document-agent-lambda-role"
  - [ ] Click "Save" button
- [ ] Configure Function Environment Variables:
  - [ ] In the "Configuration" tab, select "Environment variables"
  - [ ] Click "Edit" button
  - [ ] Add the following key-value pairs (matching your .env file):
    * OPENAI_API_KEY=[your-openai-api-key]
    * OPENAI_ORGANIZATION_ID=[your-openai-org-id]
    * AWS_S3_BUCKET=[your-document-storage-bucket]
    * DATABASE_URL=[your-dynamodb-url]
    * API_GATEWAY_URL=[your-api-gateway-url]
  - [ ] Click "Save" button
- [ ] Configure Function Concurrency:
  - [ ] In the "Configuration" tab, select "Concurrency"
  - [ ] Click "Edit" button
  - [ ] Select "Provisioned concurrency" and set value to 10 (adjust as needed)
  - [ ] Click "Save" button
- [ ] Configure VPC Access (if needed):
  - [ ] In the "Configuration" tab, select "VPC"
  - [ ] Click "Edit" button
  - [ ] Select your VPC from the dropdown
  - [ ] Select at least two subnets from different availability zones
  - [ ] Select security group that allows necessary outbound access
  - [ ] Click "Save" button

### 7.3. API Gateway Production Configuration
- [ ] Access API Gateway Console:
  - [ ] From AWS Management Console, search for "API Gateway" and select it
  - [ ] Click "Create API" button
- [ ] Create REST API:
  - [ ] Select "REST API" and click "Build" button
  - [ ] Select "New API" and enter name "AI-Document-Agent-API"
  - [ ] Select "Regional" for endpoint type
  - [ ] Click "Create API" button
- [ ] Create Resources and Methods:
  - [ ] Click "Create Resource" button
  - [ ] Enter "agent" for Resource Name and Resource Path
  - [ ] Enable CORS if needed by checking "Enable API Gateway CORS"
  - [ ] Click "Create Resource" button
  - [ ] With "agent" resource selected, click "Create Method"
  - [ ] Select "POST" from the dropdown and click checkmark
  - [ ] Select "Lambda Function" for Integration type
  - [ ] Select the Lambda function region and enter function name
  - [ ] Enable "Use Lambda Proxy integration"
  - [ ] Click "Save" button
  - [ ] Create additional resources and methods as needed (message, health, etc.)
- [ ] Configure Production Stage:
  - [ ] Click "Stages" in the left sidebar
  - [ ] Click "Create Stage" button
  - [ ] Enter "prod" for Stage name
  - [ ] Add description "Production deployment stage"
  - [ ] Click "Create" button
- [ ] Deploy API to Production:
  - [ ] Click "Resources" in the left sidebar
  - [ ] Click "Deploy API" button from the "Actions" dropdown
  - [ ] Select "prod" for Deployment stage
  - [ ] Add deployment description
  - [ ] Click "Deploy" button
- [ ] Configure Custom Domain (if needed):
  - [ ] Click "Custom Domain Names" in the left sidebar
  - [ ] Click "Create" button
  - [ ] Enter domain name (e.g., api.yourdomain.com)
  - [ ] Select TLS 1.2 for Security policy
  - [ ] Select your ACM certificate (create one if needed)
  - [ ] Click "Create domain name" button
  - [ ] Add API mapping: select API, stage, and path
  - [ ] Create CNAME record in your DNS provider pointing to the provided endpoint
- [ ] Set Up API Keys:
  - [ ] Click "API Keys" in the left sidebar
  - [ ] Click "Create API key" button
  - [ ] Enter name "AI-Document-Agent-APIKey"
  - [ ] Select "Auto Generate" for API key
  - [ ] Click "Save" button
- [ ] Create Usage Plan:
  - [ ] Click "Usage Plans" in the left sidebar
  - [ ] Click "Create" button
  - [ ] Enter name "Standard-Usage-Plan"
  - [ ] Enable throttling: Rate 10 requests per second, Burst 20 requests
  - [ ] Enable quota: 1000 requests per day (adjust as needed)
  - [ ] Click "Next" button
  - [ ] Associate API stages: select your API and prod stage
  - [ ] Click "Next" button
  - [ ] Add API keys: select the API key created earlier
  - [ ] Click "Done" button

### 7.4. Security and Compliance
- [ ] Review and Update IAM Policies:
  - [ ] Access IAM console
  - [ ] Click "Policies" in the left sidebar
  - [ ] For each role, click the role name
  - [ ] Click "Permissions" tab and review attached policies
  - [ ] Click on each policy and review permissions
  - [ ] Edit or create new policies with least privilege in mind
- [ ] Set Up AWS WAF (Web Application Firewall):
  - [ ] From AWS console, search for "WAF" and select it
  - [ ] Click "Create web ACL" button
  - [ ] Enter name "AI-Document-Agent-WAF"
  - [ ] Select regional resources
  - [ ] Associate with your API Gateway stage
  - [ ] Add managed rules:
    * AWS-AWSManagedRulesCommonRuleSet
    * AWS-AWSManagedRulesKnownBadInputsRuleSet
    * AWS-AWSManagedRulesSQLiRuleSet
  - [ ] Add rate-based rule to limit requests to 1000 per IP per 5 minutes
  - [ ] Set rule action to "Block"
  - [ ] Click "Create web ACL" button
- [ ] Configure AWS Shield (if needed for DDoS protection):
  - [ ] From AWS console, search for "Shield" and select it
  - [ ] Review pricing (Shield Advanced has additional costs)
  - [ ] Click "Subscribe to Shield Advanced" for enhanced protection
  - [ ] Select resources to protect (API Gateway, CloudFront)
  - [ ] Complete subscription process
- [ ] Implement AWS Secrets Manager:
  - [ ] From AWS console, search for "Secrets Manager" and select it
  - [ ] Click "Store a new secret" button
  - [ ] Select "Other type of secret"
  - [ ] Add key-value pairs for sensitive information (API keys, credentials)
  - [ ] Click "Next" button
  - [ ] Enter name "AI-Document-Agent-Secrets"
  - [ ] Add description and tags
  - [ ] Click "Next" button
  - [ ] Configure automatic rotation if needed
  - [ ] Complete the creation process
  - [ ] Update Lambda environment variables to retrieve secrets from Secrets Manager

### 7.5. Data Management
- [ ] Configure Data Retention Policies:
  - [ ] For S3: Access S3 console
    * Select your document bucket
    * Click "Management" tab
    * Click "Create lifecycle rule"
    * Enter rule name "Document-Retention-Policy"
    * Define scope (entire bucket or specific prefix)
    * Add transitions to move older objects to cheaper storage classes
    * Add expiration actions for documents older than required retention period
    * Click "Create rule" button
  - [ ] For DynamoDB: Access DynamoDB console
    * Select your table
    * Click "Backups" tab
    * Click "Create backup" to create manual backup
    * Enable point-in-time recovery if needed
- [ ] Set Up Database Backup Schedules:

  - [ ] Select "Start with a template" and choose "Daily-Monthly-1yr-Retention"

  - [ ] Click "Create backup plan" button
  - [ ] Click "Assign resources" button
  - [ ] Select resource types (DynamoDB tables)
  - [ ] Select specific resources to back up
  - [ ] Click "Assign resources" button
